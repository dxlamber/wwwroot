收到一个http请求后：
  1. 其路径转换为一个本机的绝对路径，并判断路径所指文件是否存在。
    a. 存在，并且不是可执行文件，将文件内容读出，作为http响应返回给客户端。
    b. 存在，是可执行文件，执行之，将执行之后返回的结果，作为http响应返回给客户端。
    c. 不存在，转第二步。
  2. 其路径在指定的web 接口的路径之下。跳转到指定的web接口目录下的指定接口，执行，将执行之后返回的结果，作为http响应返回给客户端。
    a. 类似于1.a步的处理，或者两者完全一样。
  3. 或者像nginx那样，根据文件后缀名，比如所有.php文件都交给php-perfermance去执行。



MIME的作用：
首先，我们要了解浏览器是如何处理内容的。在浏览器中显示的内容有 HTML、有 XML、有 GIF、还有 Flash ...
那么，浏览器是如何区分它们，绝对什么内容用什么形式来显示呢？答案是 MIME Type，也就是该资源的媒体类型。
媒体类型通常是通过 HTTP 协议，由 Web 服务器告知浏览器的，更准确地说，是通过 Content-Type 来表示的
web服务器使用MIME来说明发送数据的种类， web客户端使用MIME来说明希望接收到的数据种类。
具体使用：
  1. 当web服务器向浏览器发送文档时，它会给文档加上一个响应报头作为前缀，此报头包含了用于告诉浏览器如何解释文档的数据。
元数据的一个最重要的部分是最后一行中的Content-Type。它将告诉浏览器如何呈现内容。
例如，浏览器用于显示 JPEG 和 GIF 的代码是不同的。
  2. 当web服务器收到静态的资源文件请求时，依据请求文件的后缀名在服务器的MIME配置文件中找到对应的MIME Type，
再根据MIME Type设置HTTP Response的Content-Type，然后浏览器根据Content-Type的值处理文件。

通常只有一些在互联网上获得广泛应用的格式才会获得一个 MIME Type，如果是某个客户端自己定义的格式，一般只能以 application/x- 开头。
